## STEP 1 — Observation (관찰)

### 1) 현재 우리가 해결하려는 상황은 무엇인가?

- 학습자는 **4×4** 격자에 **1~16**을 쓰되, **빈칸(`0`)은 정확히 2개**인 **부분 마방진**을 과제·연습으로 다룬다.
- 완성 시 **10개 합**(행 4 + 열 4 + 대각선 2)이 모두 **34**여야 한다.
- 손 검산·코드 검증 모두에서 **“합이 맞는지”**와 **“어디가 깨졌는지”**가 동시에 필요하다.

### 2) 왜 4×4·빈칸 2개인가?

- 규칙이 많아(10선·값 집합·빈칸 개수) **검증 누락**(특히 대각선)과 **오류 국소화 실패**가 드러나기 쉽다.
- “정답 찾기”보다 **관찰 가능한 계약**으로 문제를 고정하기에 적당한 크기다.

### 3) 어떤 맥락에서 등장하는가?

- 소프트웨어 학습(과제·ECB 분류 연습) 맥락.
- Mom Test로 **솔루션 이전**에 **과거 행동·비용**을 먼저 수집한 뒤, PRD/TDD로 **번역**한다.

## Mom Test 요약 (증거 → 문제)

### 페르소나

4×4 격자, 빈칸 2개(`0`), 1~16, **합 34(10선)** 를 맞추는 **학습자**.

### 과거 사실 (인용)

| # | 사실 |
|:---:|:---|
| 1 | 지난주 과제: 빈칸 2개 채운 뒤 행·열·대각선 확인 중 **대각선 하나 빼먹어 약 20분 소모** |
| 2 | **같은 줄을 세 번** 고쳤는데 **어디가 틀린지 몰라 포기** |
| 3 | 빈칸은 **감으로** 채워 제출 → **틀림** → **다시 안 풂**(이유: 마감, 제출 시 **약 20분 남음**) |

### Mom Test 증거 (워크북·세션 3)

1. 마지막으로 빈칸 2개를 채운 뒤 **행·열·대각선(10선) 합 확인에 시간이 많이 듦**.
2. 합이 34가 안 맞을 때 **몇 번째로 고친 칸이 원인인지** 알기 어렵다.
3. (STEP 1) 같은 줄만 반복 수정 후 포기·감으로 제출.

### 표면 문제 (금지 — 기록만, 제품 방향에 쓰지 않음)

| 표면 문제 | 왜 금지인가 |
|:---|:---|
| “4×4 마방진 **자동 완성**” | 기능·알고리즘 선행 |
| “**대각선 검사 기능**이 있으면 좋다” | 솔루션 가정 |
| “**ECB/TDD/PyQt**로 만들면 된다” | 방법론·스택 선행 |

### 진짜 문제 (한 문장)

> **여러 줄·열·대각선(10선) 규칙이 동시에 걸릴 때, 내가 깨뜨린 축·칸·수정 시도를 찾지 못한 채 같은 부분만 고치다 검증에 시간을 쓰고, 확신 없이 제출한 뒤 틀려도 다시 검증할 동기가 끊긴다.**

*(세션 3 좁힌 표현: “합이 34가 안 맞았을 때, **몇 번째로 고친 칸**이 원인인지 알지 못했다.” — 위 한 문장에 포함.)*

## STEP 5 — 진짜 문제 정의

### 1) 표면 문제 정의 (잘못된 정의)

- **정의**: “\(4 \times 4\) **마방진을 만든다**.”
- **왜 잘못됐는가**
  - 값 집합(1~16)·빈칸 2개·**10선 합 34**가 빠짐.
  - 성공/실패·검증·완성/판정이 한 문장에 뒤섞임.
  - 테스트로 고정할 **입력/출력**이 없음.

### 2) 개선된 문제 정의 (정확한 정의)

**정의(관찰 가능한 계약 중심)**

> 프로그램(또는 학습자 절차)은 \(4 \times 4\) 격자에 대해, **0을 정확히 2개** 포함한 부분 배치를 입력으로 받아,  
> (a) **합의된 값·규칙**(1~16, 10선 합 34)을 만족하는 **완성 배치**를 산출하거나,  
> (b) 그러한 완성이 **없음**을 판정하며,  
> (c) 배치가 규칙을 만족하는지 **기계적으로 검증**할 수 있어야 한다.  
> 실패 시 **첫 위반 1개**(어떤 축·어떤 선이 깨졌는지)를 **결정적으로** 보고한다.

**Mom Test와의 연결**

| 진짜 문제(고통) | PRD/계약으로의 번역 |
|:---|:---|
| 10선 확인 비용·대각선 누락 | `is_magic_square` / 10선 검사 **고정 순서**(행→열→대각) |
| 어디가 틀렸는지 모름 | **첫 위반 1개** 에러·우선순위(§5.3 PRD) |
| 감으로 제출 | 입력 계약 위반 시 **조기 거부** + 명시 `message` |
| 틀린 뒤 재시도 없음 | 결정적 출력·회귀 테스트로 **재검증 비용** 감소(학습 루프) |

### 3) 이 문제의 핵심 Invariant

| ID | 불변식 | 내용 |
|:---|:---|:---|
| **I-01** | Shape | 입력은 정확히 4×4 |
| **I-02** | EmptyCount | `0`은 정확히 2개 |
| **I-03** | Range | 각 값은 `0` 또는 `1..16` |
| **I-04** | UniqNonZero | `0` 제외 중복 없음 |
| **I-05** | MissingExactly2 | 1..16 중 누락 2개(I-02~I-04 만족 시 도출) |
| **I-06** | MagicConstant | 완성 시 마방진 상수 **34** |
| **I-07** | RowSum | 4행 합 동일(완성 시 34) |
| **I-08** | ColSum | 4열 합 동일(완성 시 34) |
| **I-09** | DiagSum | 두 대각선 합 동일(완성 시 34) — *Mom Test: 누락되기 쉬운 축* |
| **I-10** | DeterministicOrderRule | 성공 출력 `[r1,c1,n1,r2,c2,n2]` 순서 결정적 |

**10선**: I-07 + I-08 + I-09가 검증 대상(행 4 + 열 4 + 대각 2).

### 4) 우리가 실제로 훈련하려는 사고 능력

- **요구를 계약으로 바꾸기**: “만든다” → 입력/출력/실패/첫 위반.
- **불변식 중심 설계**: 10선·빈칸 2·34를 테스트로 고정.
- **검증과 생성 분리**: 완성 탐색과 `is_magic_square` 판정 분리.
- **오류 국소화**: 같은 줄만 고치기 전에 **어느 축이 깨졌는지** 먼저 특정.
