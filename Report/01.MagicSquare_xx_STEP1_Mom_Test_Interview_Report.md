## 3. Mom Test 워크북 (이 인터뷰만 근거)

### 3.1 페르소나

4×4 **부분 마방진**(빈칸 2개)을 과제·연습으로 손/코드로 다루는 **학습자**. 행·열·대각선 규칙은 알지만, 검산 중 **어느 제약이 깨졌는지 위치를 못 잡으면** 같은 부분만 고치다 시간을 쓰고, 확신 없이 제출하는 경험이 있음.

### 3.2 인터뷰 답변 (과거 사실 — 직접 인용)

- “같은 줄을 세 번 고쳤는데 어디가 틀린지 몰라 포기했다.”
- “빈칸은 감으로 채워서 냈고, 틀렸다.”
- “틀린 뒤에는 마감 때문에(20분 남았어도) 다시 안 풀고 그냥 두었다.”

### 3.3 표면 문제 (금지 — 솔루션·기능 섞임)

아래는 인터뷰 중 **기록만** 하고 제품·스펙 방향으로 쓰지 않음.

| 표면 문제 (금지 예) | 왜 금지인가 |
|:---|:---|
| “4×4 마방진 **자동 완성**이 필요하다.” | 기능·알고리즘이 이미 포함 |
| “**대각선 검사 기능**이 있으면 좋겠다.” | 구현 항목으로 바로 연결 |
| “**ECB/TDD로** MagicSquare를 만들면 된다.” | 방법론·프로젝트명 = 솔루션 |

### 3.4 진짜 문제 (한 문장)

> **여러 줄·열·대각선 규칙이 동시에 걸릴 때, 내가 깨뜨린 게 어디인지 찾지 못한 채 같은 부분만 고치다 시간을 쓰고, 확신 없이 제출한 뒤 틀려도 다시 검증할 동기가 마감·피로로 끊긴다.**

## 4. 관찰·모순 (인사이트)

| 관찰 | 시사점 |
|:---|:---|
| 대각선 검사 누락으로 **20분 소모** | “합 맞추기”가 **어떤 축(행/열/대각)**까지 포함인지 실행 시 빠지기 쉬움 |
| 같은 줄만 **3회 수정** 후 포기 | 오류 **국소화 실패** → 수정 방향이 잘못된 축에 고정됨 |
| 마감 **20분 남음**에도 감으로 제출 | 시간 부족만이 아니라 **혼란·피로·재시도 회피** 가능성 (질문 9로 미확인) |
| 틀린 뒤 **재시도 없음** | 학습 루프가 “피드백 → 수정”으로 이어지지 않음 |

---

## 5. 미완료·후속 인터뷰

- **질문 9**: 제출 시 마감 20분 **남은 상태**에서, 그 시간에 마방진 외에 **실제로 한 일** (다른 과제, 휴식 등).
- **표면 문제 수집**: 응답자가 **스스로** 말한 “~ 있으면 좋겠다”류 문장이 이번 세션에는 없음 — 필요 시 별도 세션.
